# 编译器架构后端重构计划

## 目标
重构编译器架构后端，仅支持以下目标：
- Zig（先编译为zig源码，再生成native、wasm）
- Python（直接生成python源码）
- Java（直接生成Java字节码）
- C#（直接生成.net字节码）
- JavaScript（生成Typescript源码再转译js）

## 实现策略

1. **清理阶段**：先清理不需要的目标平台和后端代码
2. **实现阶段**：按照优先级顺序实现各个后端
3. **集成阶段**：将所有后端集成到代码生成器接口
4. **测试阶段**：全面测试所有后端的功能

## 技术要点

- **Zig 后端**：利用现有的实现，确保支持 native 和 wasm 目标
- **Python 后端**：生成清晰可读的 Python 源码
- **Java 后端**：使用适当的库生成 Java 字节码
- **C# 后端**：使用适当的库生成 .NET 字节码
- **JavaScript 后端**：生成 TypeScript 源码，然后使用 tsc 转译为 JavaScript

## 风险评估

- **Java 字节码生成**：可能需要使用第三方库，增加依赖
- **.NET 字节码生成**：同样可能需要第三方库
- **TypeScript 转译**：需要确保 TypeScript 编译器可用
- **测试覆盖**：需要为每个后端编写充分的测试用例

## 预期成果

- 一个精简的编译器后端架构，只包含所需的目标平台
- 各个后端都能正确编译 X 语言代码
- 生成的代码质量良好，功能正确
- 完整的测试覆盖
