# 文档构建系统转换计划 - 从Jekyll到mdbook

## 项目概述

本计划旨在将X编程语言的文档从Jekyll构建系统转换为mdbook构建系统，以提供更简洁、更专注于技术文档的构建体验。

## 转换步骤

### [x] 任务1: 创建mdbook基本结构
- **优先级**: P0
- **依赖**: 无
- **描述**:
  - 在docs目录中创建src子目录
  - 创建book.toml配置文件
  - 创建src/SUMMARY.md文件作为文档目录
- **成功标准**:
  - 基本目录结构已创建
  - book.toml配置文件已配置
  - SUMMARY.md文件已创建并包含基本结构
- **测试要求**:
  - `programmatic` TR-1.1: 目录结构正确创建 ✓
  - `human-judgement` TR-1.2: 配置文件格式正确 ✓
- **注意事项**: 保持现有文件不变，仅添加新的mdbook结构

### [x] 任务2: 生成SUMMARY.md文件
- **优先级**: P0
- **依赖**: 任务1
- **描述**:
  - 分析现有文档结构
  - 根据sidebar.html内容创建SUMMARY.md
  - 确保所有现有章节都包含在目录中
- **成功标准**:
  - SUMMARY.md文件包含所有现有章节
  - 目录结构清晰，层次分明
  - 链接格式正确
- **测试要求**:
  - `programmatic` TR-2.1: SUMMARY.md文件格式正确 ✓
  - `human-judgement` TR-2.2: 目录结构与现有文档一致 ✓
- **注意事项**: 确保所有文件路径正确，特别是相对路径

### [x] 任务3: 移动和整理文档文件
- **优先级**: P0
- **依赖**: 任务2
- **描述**:
  - 将所有Markdown文件移动到src目录
  - 移除Jekyll特定的front matter
  - 调整文件中的链接路径
- **成功标准**:
  - 所有Markdown文件已移动到src目录
  - 所有文件已移除Jekyll front matter
  - 内部链接已更新为mdbook格式
- **测试要求**:
  - `programmatic` TR-3.1: 所有文件已正确移动 ✓
  - `human-judgement` TR-3.2: 链接路径正确 ✓
- **注意事项**: 保持文件命名一致，确保不丢失任何内容

### [x] 任务4: 配置book.toml文件
- **优先级**: P1
- **依赖**: 任务3
- **描述**:
  - 配置book.toml文件，设置标题、作者、语言等
  - 配置主题和插件
  - 设置输出目录
- **成功标准**:
  - book.toml配置完整
  - 主题设置合理
  - 插件配置正确
- **测试要求**:
  - `programmatic` TR-4.1: book.toml格式正确 ✓
  - `human-judgement` TR-4.2: 配置选项合理 ✓
- **注意事项**: 参考mdbook文档获取最佳配置实践

### [x] 任务5: 测试构建过程
- **优先级**: P0
- **依赖**: 任务4
- **描述**:
  - 运行mdbook build命令
  - 检查构建输出
  - 验证所有页面是否正确生成
- **成功标准**:
  - 构建过程无错误
  - 所有页面已生成
  - 链接和导航正常工作
- **测试要求**:
  - `programmatic` TR-5.1: 构建过程成功完成 ✓
  - `human-judgement` TR-5.2: 生成的文档结构正确 ✓
- **注意事项**: 检查构建日志，确保没有警告或错误

### [x] 任务6: 优化和调整

- **依赖**: 任务5

- **成功标准**:

### [x] 任务7: 清理和文档
- **优先级**: P2
- **依赖**: 任务6
- **描述**:

  - 更新发布说明
  - 编写mdbook使用指南
- **成功标准**:
  - 移除不必要的Jekyll文件
  - 发布说明已更新
  - 提供清晰的使用指南
- **测试要求**:
  - `programmatic` TR-7.1: 清理工作完成 ✓
  - `human-judgement` TR-7.2: 文档说明清晰 ✓
- **注意事项**: 保留必要的文件，确保不影响其他功能
