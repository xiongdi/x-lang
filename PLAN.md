## 实现顺序建议

按依赖顺序，应该按以下顺序工作：

> **1. 第一阶段 → 2. 第二阶段 → 3. 第三阶段 → 4. 第四阶段 → 5. 第五阶段 → 6. 第六阶段 → 7. 第七阶段 → 8. 第八阶段**

这个顺序保证每次都能得到一个可工作的系统，依赖清晰，风险可控：

1. **先完成第一阶段**（前端）→ 所有语法都能正确解析
2. **再完成第二阶段**（类型检查）→ 所有语法都能正确类型检查
3. **再完成第三阶段**（IR + Perceus）→ 完整流水线到 LIR
4. **再完成第四阶段**（Zig 后端）→ Zig 后端能从 LIR 生成完整代码
5. **再完成第五阶段**（解释器）→ `x run` 能运行所有示例
6. **再逐步完善其他后端**（按优先级：Rust → TypeScript → LLVM → Native → 其他）
7. **最后完善标准库和工具链**

## 优先级说明

根据 **DESIGN_GOALS.md**，优先级为：

1. **可读性第一** - 语法解析必须正确，错误信息可读
2. **类型安全** - 完整类型检查比多个后端更重要
3. **内存安全** - Perceus 必须正确工作
4. **Zig 为先** - Zig 后端是当前最成熟的，应优先完成
5. **完整标准库** - 语言必须能用，需要基本标准库
6. **多后端** - 最后扩展到其他目标平台
