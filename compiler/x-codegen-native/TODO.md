# X 语言 Native 后端实现 TODO

Native 后端目标：**LIR → 汇编文本 → 汇编器 → 链接器 → 原生可执行文件**。

## 架构设计

### Windows 平台

```
LIR (XIR) → NativeBackend → MASM 汇编 (.asm)
                                    ↓
                                  ml64 (随 Visual Studio 提供)
                                    ↓
                              目标文件 (.obj)
                                    ↓
                                link.exe (随 Visual Studio 提供)
                                    ↓
                              可执行文件 (.exe)
```

**优势**：
- 🚀 开发速度快：不需要手工编码每条指令
- ✅ 成熟稳定：MASM 正确处理所有 x86_64 指令编码
- 🔧 易于调试：汇编文本可直接查看和验证
- 📦 **零额外依赖**：ml64 和 link.exe 随 Visual Studio/Windows SDK 提供

### Linux/macOS 平台

```
LIR (XIR) → NativeBackend → GAS/NASM 汇编 (.s/.asm)
                                    ↓
                              as / nasm
                                    ↓
                              目标文件 (.o)
                                    ↓
                                ld / gcc
                                    ↓
                              可执行文件
```

## 设计目标

- ⚡ **快速编译**：毫秒级编译速度，适合快速开发迭代
- 📦 **最小依赖**：只需 NASM + GoLink（Windows）/ ld（Linux）
- 🐛 **调试优先**：完整的调试信息支持
- 🔁 **自举基础**：为 X 编译器自编译提供基础

## 关键设计决策

1. **Windows 使用 MASM + link.exe**：随 Visual Studio 提供，无需额外安装
2. **Linux/macOS 使用 GAS + ld**：系统自带，无需额外安装
3. **先做源码级调试，不做优化**：优先保证正确性和编译速度，性能其次
4. **线性扫描寄存器分配**：简单、快速、适合我们的设计目标
5. **动态链接系统 C 库**：不用自己实现 printf/malloc/free 等基础函数
6. **一步一个脚印**：每个阶段都有可运行的输出，随时可以停下来用
7. **优先支持 x86_64 Windows**：因为当前开发环境就是 Windows，先能在本机测试
