## 附录C：类型转换表

X语言支持以下类型转换：

### 隐式类型转换

| 源类型 | 目标类型 | 转换规则 |
|-------|---------|----------|
| `integer 8` | `integer 16/32/64/128` | 无符号扩展 |
| `integer 16` | `integer 32/64/128` | 无符号扩展 |
| `integer 32` | `integer 64/128` | 无符号扩展 |
| `integer 64` | `integer 128` | 无符号扩展 |
| `integer` (任意) | `float 32/64` | 数值转换 |
| `unsigned integer` (任意) | `float 32/64` | 数值转换 |
| `float 32` | `float 64` | 精度扩展 |
| `Never` | 任意类型 | 永远不会发生的转换 |

### 显式类型转换

| 转换函数 | 说明 | 示例 |
|---------|------|------|
| `to_integer(value)` | 转换为整数 | `to_integer(3.14) // 3` |
| `to_float(value)` | 转换为浮点数 | `to_float(42) // 42.0` |
| `to_string(value)` | 转换为字符串 | `to_string(123) // "123"` |
| `to_boolean(value)` | 转换为布尔值 | `to_boolean(0) // false` |
| `as<T>(value)` | 类型断言 | `as<String>(value)` |

### 类型提升规则

在混合类型的表达式中，X语言会自动执行类型提升：

1. 整数运算：结果类型为参与运算的最宽整数类型
2. 整数与浮点数运算：结果类型为浮点数类型
3. 浮点数运算：结果类型为参与运算的最宽浮点数类型

## 附录E：常见问题

### 语法问题

**Q: 如何定义一个函数？**
A: 使用 `function` 关键字定义函数：
```x
function add(a: integer, b: integer) -> integer {
    return a + b
}
```

**Q: 如何定义一个类？**
A: 使用 `class` 关键字定义类：
```x
class Person {
    private name: string
    private age: integer
    
    function constructor(name: string, age: integer) {
        this.name = name
        this.age = age
    }
    
    public function get_name() -> string {
        return this.name
    }
}
```

**Q: 如何使用模式匹配？**
A: 使用 `when` 关键字进行模式匹配：
```x
when value {
    Some(x) => println("Found: {x}")
    None => println("Not found")
}
```

### 类型系统问题

**Q: X语言是否有null？**
A: X语言没有null，使用 `Option<T>` 类型表示可能缺失的值。

**Q: X语言是否有异常？**
A: X语言没有异常，使用 `Result<T, E>` 类型表示可能失败的操作。

**Q: 如何处理错误？**
A: 使用 `Result<T, E>` 类型和模式匹配处理错误：
```x
match read_file("config.txt") {
    Ok(content) => println(content)
    Err(error) => println("Error: {error}")
}
```

### 效果系统问题

**Q: 如何定义异步函数？**
A: 使用 `async` 关键字定义异步函数：
```x
async function fetch_data(url: string) -> string {
    let response = await http_get(url)
    return response.body
}
```

**Q: 如何处理并发？**
A: 使用 `together` 关键字并行执行多个操作：
```x
together {
    let data1 = fetch_data("url1")
    let data2 = fetch_data("url2")
} returns (data1, data2)
```

### 性能问题

**Q: X语言的性能如何？**
A: X语言设计时考虑了性能因素，通过以下方式提高性能：
- 基于LLVM的编译优化
- Perceus所有权系统减少内存分配
- 静态类型检查减少运行时开销
- 惰性求值和尾递归优化

**Q: 如何优化X语言程序？**
A: 优化建议：
- 使用不可变变量（默认）
- 避免不必要的克隆操作
- 使用管道运算符组合函数
- 利用惰性求值减少计算
- 合理使用类型注解帮助编译器优化

### 工具链问题

**Q: 如何编译X语言程序？**
A: 使用X语言编译器：
```bash
x compile main.x
```

**Q: 如何运行X语言程序？**
A: 使用X语言运行时：
```bash
x run main.x
```

**Q: 如何测试X语言程序？**
A: 使用X语言测试工具：
```bash

```

### 与其他语言的比较

**Q: X语言与Rust的区别？**
A: X语言借鉴了Rust的许多设计理念，但在语法和易用性上做了改进：
- 更简洁的语法
- 更强大的类型推断
- 内置的效果系统
- 更友好的错误处理

**Q: X语言与JavaScript的区别？**
A: X语言与JavaScript的主要区别：
- 静态类型系统
- 更严格的错误处理
- 内存安全保证
- 更强大的类型系统

**Q: X语言与Python的区别？**
A: X语言与Python的主要区别：
- 静态类型检查
- 编译型语言

## 附录H：编码规范

### 命名约定
- 变量和函数：小驼峰命名法 (`camelCase`)
- 类型和模块：大驼峰命名法 (`PascalCase`)
- 常量：全大写加下划线 (`UPPER_CASE`)

### 代码风格
- 缩进：4个空格
- 行宽：不超过100个字符
- 空行：函数之间用空行分隔
- 注释：使用 `//` 进行单行注释

### 最佳实践
- 优先使用不可变变量
- 明确处理 `Option` 和 `Result` 类型
- 使用管道运算符提高代码可读性
- 为公共API添加类型注解
- 编写单元测试
